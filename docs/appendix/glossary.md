# 附录 C - 技术词汇表

## A

### Agent
智能代理，能够自主感知环境、做出决策并执行动作的 AI 系统。

### Agentic
具有 Agent 特性的，强调 AI 的自主性和行动能力。

### API (Application Programming Interface)
应用程序编程接口，软件之间通信的约定。

## B

### Backpressure
背压，响应式编程中处理生产速度 > 消费速度的机制。

### ByteAgent
字节级 Agent，能直接操作字节数据的代理。

## C

### Chunking
分块，将长文本切分成小块以便处理和索引。

### Compaction
压缩，在上下文窗口接近上限时合并/摘要历史内容。

### Context Window
上下文窗口，LLM 一次能处理的最大 token 数量。

## D

### Docusaurus
Facebook 开源的文档网站生成工具。

## E

### Embedding
嵌入，将文本转换为向量表示的过程。

### EventBus
事件总线，发布-订阅模式的事件分发机制。

## F

### Few-shot
少样本学习，用少量示例教会模型完成任务。

### Flush
刷新，将内存中的数据持久化到存储。

### Flyway
数据库迁移工具，管理数据库版本演进。

### Function Calling
函数调用，LLM 调用外部函数/工具的机制。

## G

### Gating
门控，技能激活前的环境检查机制。

### Grounding
接地，确保 AI 输出基于事实而非幻觉。

## H

### HITL (Human-in-the-Loop)
人在回路，关键决策需要人工确认的机制。

## L

### LLM (Large Language Model)
大语言模型，如 GPT、Claude、LLaMA 等。

### Lazy Loading
延迟加载，按需加载而非一次性全部加载。

## M

### MCP (Model Context Protocol)
模型上下文协议，Anthropic 提出的 AI 工具互操作标准。

### Mermaid
文本转图表的工具，用代码生成流程图、时序图等。

### Multi-Agent
多代理，多个 Agent 协作完成复杂任务。

## O

### OpenAI API
OpenAI 提供的 LLM API，已成为事实上的行业标准。

### Overlap
重叠，文本分块时相邻块之间的共享部分。

## P

### pgvector
PostgreSQL 的向量扩展，支持向量存储和检索。

### Prompt
提示词，给 LLM 的输入指令。

### PWA (Progressive Web App)
渐进式 Web 应用，支持离线和安装的 Web 应用。

## R

### ReAct (Reasoning + Acting)
推理+行动，AI Agent 的核心循环模式：Think → Act → Observe。

### RPC (Remote Procedure Call)
远程过程调用，跨进程/网络调用函数的机制。

## S

### Sandbox
沙箱，隔离的执行环境，限制危险操作。

### SES (Server-Sent Events)
服务器推送事件，服务器向客户端单向推送的协议。

### Session
会话，用户与 Agent 的一次连续交互。

### Skill
技能，Agent 的专业能力模块。

### SSE
见 Server-Sent Events。

### SubAgent
子代理，主 Agent 派生的辅助 Agent。

### System Prompt
系统提示，定义 Agent 行为和规则的顶层指令。

## T

### Token
词元，LLM 处理文本的基本单位。

### Tool
工具，Agent 可以调用的外部功能。

### ToolDispatcher
工具调度器，分发工具调用请求的组件。

## V

### Vector Search
向量检索，基于语义相似度的搜索方式。

### Virtual Threads
虚拟线程，JDK 21+ 提供的轻量级线程。

## W

### WebFlux
Spring 的响应式 Web 框架。

### WebSocket
全双工通信协议，支持客户端和服务器双向实时通信。

### Workspace
工作区，Agent 操作文件和数据的根目录。
