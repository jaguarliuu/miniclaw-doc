# 附录 A - AI 编程提效技巧

:::info 定位
本附录介绍如何利用 AI 辅助编程，提高开发效率。
:::

## AI 编程的本质

不是"替代程序员"，而是"协作编程"。

### 协作思维 vs 替代思维

| 替代思维 | 协作思维 |
|---------|---------|
| "AI 帮我写代码" | "AI 帮我想清楚要写什么" |
| AI 是工具 | AI 是搭档 |
| 复制粘贴 | 理解 + 验证 + 改进 |

## AI 辅助编程的最佳实践

### 1. 精准描述需求

❌ 差的提示：
```
帮我写一个登录功能
```

✅ 好的提示：
```
用 Spring Boot + JWT 实现用户登录功能：
1. 用户名/密码登录
2. 登录成功返回 JWT token
3. 密码用 BCrypt 加密
4. 登录失败返回 401
```

### 2. 让 AI 理解上下文

- 提供相关代码片段
- 说明技术栈和依赖版本
- 描述项目结构和命名规范

### 3. 分步骤拆解任务

大任务 → 小任务 → 让 AI 逐个完成

### 4. 审查 AI 生成的代码

- **永远不要**盲目复制 AI 代码
- 理解每一行的作用
- 测试边界情况
- 检查安全漏洞

## 常见 AI 编程反模式

1. **不加思考直接复制** — 必须理解
2. **过度依赖 AI** — 核心逻辑自己写
3. **忽略代码审查** — AI 也会犯错
4. **不测试就上线** — 必须验证

## 推荐的 AI 编程工作流

```
1. 自己先想清楚需求和设计
2. 让 AI 帮你生成代码框架
3. 自己填充核心逻辑
4. 让 AI review 你的代码
5. 根据反馈改进
6. 编写测试验证
```
